Introduction

This Python script, named habit_tracker.py, is designed to interact with the Pixela API, which allows users to create graphs to track their habits or activities over time. Let's break down the main components of this script:

  1. Importing Libraries: The script imports necessary libraries such as requests for making HTTP requests and datetime for working with dates.
  1. Constants Definition: The script defines constants such as USERNAME, TOKEN, and GRAPH_ID. These are essential for authenticating with the Pixela API and identifying the user and the graph to be manipulated.
  1. User Registration: The script includes code (commented out) to register a new user with the Pixela service. It sends a POST request to the Pixela endpoint with user parameters including the username, token, and agreement to terms of service.
  1. Graph Creation: Similarly, there's code (commented out) for creating a new graph. This involves specifying details such as the graph's ID, name, unit (e.g., kilometers), type (e.g., float), and color.
  1. Pixel Creation: The script prompts the user to input the quantity of an activity for the current day (in this case, kilometers cycled). It then sends a POST request to the Pixela endpoint to create a new pixel on the graph for today's date with the specified quantity.
  1. Pixel Update and Deletion: There are sections of code (commented out) for updating and deleting pixels on the graph. This involves sending PUT or DELETE requests to the appropriate endpoints with the updated pixel data or simply the date of the pixel to be deleted.

Overall, this script provides a basic framework for interacting with the Pixela API to track habits or activities over time, specifically focusing on cycling in this example. Users can extend this script to track other habits by modifying the prompts and data sent to the Pixela API.
