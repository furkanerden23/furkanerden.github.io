Flex Direction

Row

Default is row

Elements are lined up on a row

	flex-direction: row;
}

Column

Elements are lined up on a column

	flex-direction: column;
}

Flex Layout

Order

Higher order gets closer to the right side in the row direction

.container .yellow{
	order: 14;
}

Wrap

Are items gonna run out from screen or not?

.container {
	flex-wrap: nowrap;  /* Run out from screen */
	flex-wrap: wrap;    /* Do not run out and use next row */
}

Justify

Row starts from the left or right side

.container {
	justify-content: flex-start;   /* Row starts from the left side */
	justify-content: flex-end;     /* Row bunched up all to the right side */
	justify-content: center;       /* Row centered */
	justify-content: space-between;/* Row centered but put space between children and make them touch to sides */
	justify-content: space-around; /* Same as space-between but puts the same space to the sides too */
  justify-content: space-evenly; /* Gaps are equal */
}

Align

ALIGN ITEMS

which usually sets the distribution along the cross-axis

.container {
	align-items: flex-start;       /* Row located top side of the container */
	height: 70vh;                  /* You need hight to align, vh:viewport height */

	align-items: flex-end;         /* Row located bottom side of the container */
	align-items: flex-center;      /* Row located center of the container */
	align-items: flex-baseline;    /* Row located base side of the container */
	align-items: flex-stretch;     /* Row stretched from top to the bottom of the container */
}

Flex Sizing

Priority list to determine flex item sizes

First, we need flex container

.container {
	display: flex;
	gap: 10px;
}

WIDTH

If there is enough space, items is gonna be 100 px wide max.

If there is no space, it will dynamically shrink the items

.item {
	width: 100px;
}

FLEX BASIS

flex-basis is more important than width, so width will be ignored

Items are gonna be maximum of 200px wide

Shrink is ON

.item {
	width: 100px;
	flex-basis: 200px;
}

MIN WIDTH - MAX WIDTH

This is the most important one, it ignores all above

Items are gonna be maximum of 100px wide

.item {
	flex-basis: 200px;
	max-width: 100px;
}

Items are gonna be minimum of 300 px wide

No more shrink

.item {
	flex-basis: 200px;
	min-width: 300px;
}

TURN OFF SHRINK AND GROW

Items are gonna be 100px wide

.item {
	flex-basis: 100px;
	flex-grow: 0:
	flex-shrink: 0;
}

Items are gonna be min 100px wide

.item {
	flex-basis: 100px;
	flex-grow: 1:
	flex-shrink: 0;
}

Items are gonna be max 100px wide

.item {
	flex-basis: 100px;
	flex-grow: 0:
	flex-shrink: 1;
}

No base size, go smaller and wider as much as possible

.item {
	flex-basis: 100px;
	flex-grow: 1:
	flex-shrink: 1;
}

Make everything equal width

.item {
	flex-basis: 0;
	flex-grow: 1:
	flex-shrink: 1;
}

SHORTCUT

Grow ON, shrink ON, equal width for elements ON

Those three are the same


.item {
	flex-basis: 0;
	flex-grow: 1:
	flex-shrink: 1;
}

.item {
	flex: 1 1 0;  /* grow shrink basis */
}

.item {
	flex: 1;
}

.item-two is gonna double the width of .item-one

.item-one {
	flex: 1;
}

.item-two {
	flex: 2;
